# 기술 블로그 화면설계서 v2 (초안)

## 0. 문서 목적
이 문서는 개발 착수 전 개념을 정리하고, 화면 레이아웃과 동작 규칙을 정의하기 위한 화면설계서이다.

---

## 1. 용어 정의

### 1.1 태그 분류
- 프라이머리태그: 포스트의 첫 번째 태그
- 시리즈태그: `#series-{seriesid}` 형태의 internal tag
- 세컨드태그: 프라이머리태그/시리즈태그를 제외한 나머지 태그

### 1.2 식별자
- `tagid`: Ghost tag slug
- `seriesid`: `#series-{seriesid}`에서 `{seriesid}` 부분

---

## 2. 공통 정책

### 2.1 페이지네이션
- 공통: `posts_per_page = 10`
- 리스트 화면은 페이지네이션을 기본 적용한다.

### 2.2 광고
- 공통: `ad_interval = 5`
- 10개 포스트 기준 화면 블록:

```text
5 posts + 1 ad + 5 posts + 1 ad
= 10 posts + 2 ads
```

- 페이지네이션 계산은 포스트 개수 기준으로 처리한다.
- 광고는 리스트 사이에 렌더링 블록으로 삽입한다.

### 2.3 빈 데이터 처리
- 0건: `nothing happened` (중앙정렬 + 이탤릭)
- featured만 1건 존재하고 나머지가 없으면 posts 섹션은 표시하지 않는다.

---

## 3. Navigation 화면 설계

### 3.1 헤더 항목 순서(Desktop)

```text
Logo > #A > #B > #C > search button > theme toggle > sign in > subscribe
```

### 3.2 항목 정의

#### #A: 프라이머리태그 4개
- Tech Docs
- Tech Gear
- UI & UX
- Readium

동작:
- 클릭 시 `/tag/{tagid}` 이동

#### #B: +more
- 클릭 시 `/categories` 이동

#### #C: Series
- 클릭 시 `/series` 이동

### 3.3 Search / Theme / Member
- Search: `/search` 페이지 이동
- Theme Toggle: 헤더 고정 유지
- Sign in / Subscribe: Wave 기본 멤버십 동작 준용

### 3.4 시리즈 라우팅 정책
- 시리즈 소스는 `#series-*` internal tag
- `/series`는 시리즈태그 목록 허브
- 시리즈 상세 진입은 Ghost 기본 internal tag URL 사용
  - `/tag/hash-series-{seriesid}/`
- `/series/{seriesid}` 커스텀 라우트는 사용하지 않음

## 4. 화면별 레이아웃 정의

## 4.1 Tag 페이지 (`/tag/{tagid}`)

### 데이터 규칙
- featured posts:
  - 해당 tag 포스트 중 `feature this post=true` 최신 1개
  - 없으면 해당 tag 최신 1개
- posts:
  - featured posts 제외 나머지 포스트

### 레이아웃

```text
   -----------------------------------------------------------
	[  tag image] <h1> Name 
	[           ] description
	[           ] 
	-----------------------------------------------------------
	featured posts
	
	[ post image] date `프라이머리태그`
	[           ] <h1>포스트 제목
	[           ] 포스트 내용....................................
	[           ] .............................................
	-----------------------------------------------------------
	
	posts

	[ post image] date `프라이머리태그`
	[           ] <h1>포스트 제목
	[           ] 포스트 내용....................................
	[           ] .............................................
	
	[ post image] date `프라이머리태그`
	[           ] <h1>포스트 제목
	[           ] 포스트 내용....................................
	[           ] .............................................
	
	[ post image] date `프라이머리태그`
	[           ] <h1>포스트 제목
	[           ] 포스트 내용....................................
	[           ] .............................................

                    ........
	-----------------------------------------------------------
                    < 1 2 3 4 5 >
```

## 4.2 카테고리 페이지 (`/categories`)

### 데이터 규칙
- 카테고리 = 프라이머리태그
- featured categories:
  - 가장 최신 포스트를 가진 카테고리 1개
- categories:
  - featured categories 제외 나머지
